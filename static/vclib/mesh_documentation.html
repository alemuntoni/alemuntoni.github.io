Mesh Documentation

VCLib provides a very generic Mesh class that can be highly customized.

To explain how this customization is possible, we first need to introduce what are the main characteristics of a VCLib Mesh:

  • Components: a Component represent a characteristic of something else, and it contains

    data structure representing that characteristic. Examples of Components are Coordinate, BoundingBox, Color, etc. Components may be part of an Element or a Mesh;

  • Elements: an Element is a entity that is composed of one or more Components.

    Examples of Elements are Vertex, Face, …. When an Element is an argument of a Mesh data structure, the Mesh automatically creates a Container of Elements;

  • Element Containers: a Container is a collection of Elements of the same type.

Generally speaking, the Mesh class will inherit from Element Containers of Elements and Components, exposing their public member functions. Elements are themselves composed of a set of Components.
