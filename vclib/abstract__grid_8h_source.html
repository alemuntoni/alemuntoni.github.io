39// Developer Documentation
40// A class that derives from an AbstractGrid must store, in some way, an association between a
41// GridCell coordinate (a Point of unsigned integers) and elements of type ValueType. Each cell may
42// contain more than one element value, and each element value may be stored in more than one cell
43// if it intersects with more than one cell.
44//
45// This class is made using CRTP static polymorphism
46// (https://en.wikipedia.org/wiki/Curiously_recurring_template_pattern).
47// The last template parameter, is the Derived class that derives from this grid.
48// This allows to avoid calling virtual member functions, which would be called very often and may
49// cause a significant overhead during queries.
50//
51// What you need to do is to declared your Grid Data Structure as follows:
52//
53// template<typename GridType, typename ValueType>
54// class MyDSGrid :
55// public AbstractGrid<GridType, ValueType, MyDSGrid<GridType, ValueType>>
56// { ... };
57//
58// Make sure that your class:
59// - declares the AbstractGrid class as friend (if one of the following member functions is
60// protected/private);
61// - implements iterators Iterator and ConstIterator that iterate over pairs of
62// <KeyType, ValueType> (see below for more details)
63// - implements the following member functions:
64// - std::pair<Iterator, Iterator> valuesInCell(const KeyType) -> returns a pair of begin-end
65// iterators that will allow to cycle over the elements contained in a cell.
66// this function must be overloaded both in const and non-const versions, returning proper
67// const or non-const iterators.
68// - bool insertInCell(const KeyType&, const ValueType&) -> inserts the element in a
69// Grid Cell. The returned type tells if the element has been inserted (in some implementations,
70// insertion could not happen, e.g. when duplicates are not allowed).
71// - bool eraseInCell(const KeyType&, const ValueType&) -> erases all the element equal to the
72// given element from the given Grid Cell.
73// the returned type tells if the element has been erased from the given cell.
74// - Iterator end() -> classic member function that returns the end iterator
75//
76// Iterators of your class must:
77// - Iterate over pairs of <KeyType, ValueType&>:
78// - operator*() returns a pair having first member that is the key, and second member that is a
79// reference to the value
80// - operator->() return a pointer to the same pair
81// - operator+() and operator+(int), that moves the iterator in the same cell if there are more
82// values in it, or moves in the next cell-element
83//
84// You can reimplement in your derived class all the member functions implemented in this class that
85// may take advantage of a speedup depending on the internal storage.
86// You can also hide all the member functions implemented in this class that do not apply to your
87// derived class.
